## CROSS-CUTTING OBSERVATIONS

1. **Dual screen versions**: HomeScreenV2 + WelcomeScreen + WelcomeScreenV2 all exist. ResultsScreen (V1) + ResultsScreenV2 both exist. The routing in App.jsx determines which is active. The older screens are not deleted — potential routing confusion.

2. **Paywall inconsistency**: Some screens use `usePlan(email)` (HomeScreenV2), others use `usePaywall(email)` (most screens). These may produce different results depending on implementation.

3. **Shoot Mode role naming**: SetupSheetScreen mode picker labels roles as "Photographer / Assistant / Learning" but ShootModeScreen's MODE_META labels them "Narrated / Brief / Learning". Same roles, different display names.

4. **Stubs throughout Settings**: Help & FAQ, Contact support, Rate NGW, Password change, Delete account, Manage billing, Diagram style ("Blueprint" hardcoded) are all non-functional stubs.

5. **Sort button stub**: SavedSetupsScreen has a "Recent" sort button with no onClick handler — sort is not functional.

6. **Non-deterministic data**: MyKitScreen's `recipesUsingLight` falls back to `Math.random()` for light-to-recipe count when no match found.

7. **Pattern vs modifier naming**: ReferenceEvalScreen includes "beauty_dish" in the pattern options list (it's a modifier, not a lighting pattern).

8. **Static stage card**: HomeScreenV2 stage card shows hardcoded "Rembrandt / 94% / Side Key / Triangle Shadow / Catchlight" chips — these are static even when derived from real data, as the chips never update from actual result.

9. **Add Light stub**: RoomPlannerScreen "Add Light" CTA has no onClick — not wired.

10. **Shot Match routing gap**: ShotMatchScreen "Upgrade to Studio" navigates to screen 'upgrade' — this route may not be defined in App.jsx routing.

11. **LabScreen inline components**: ZoomableLightbox and AuthImage are defined inside LabScreen.jsx rather than extracted — they are large components that could be shared.

12. **ResultsScreenV2 flagged too busy**: Per project memory, ResultsScreenV2 is noted as too busy and needs a simplification pass.
